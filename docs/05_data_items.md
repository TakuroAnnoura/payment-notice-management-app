# データ項目 v1

## 1. データ項目整理の目的

本書では、支払対象者一覧の取込、支払通知書の作成、確認、エラー管理、作業ログ管理に必要なデータ項目を整理する。

初期開発では、以下のデータを主に扱う。

* 支払対象データ
* エラーデータ
* 作業ログ
* 担当者データ

---

## 2. 支払対象データ

支払通知書を作成する対象となる1件ごとのデータ。

| 項目名      | 英語名候補                  | データ型候補 | 必須 | 内容                  |
| -------- | ---------------------- | ------ | -- | ------------------- |
| ID       | id                     | 整数     | 必須 | アプリ内部で使用する一意の番号     |
| 発行番号     | issue_number           | 文字列    | 必須 | 支払通知書を識別する番号        |
| 会社名      | company_name           | 文字列    | 必須 | 支払先の会社名             |
| 住所       | address                | 文字列    | 必須 | 支払先の住所              |
| 事業者登録番号  | registration_number    | 文字列    | 任意 | 適格請求書発行事業者の登録番号     |
| 支払金額     | payment_amount         | 数値     | 必須 | 支払通知書に記載する金額        |
| 支払日      | payment_date           | 日付     | 必須 | 支払予定日または支払日         |
| 内訳       | description            | 文字列    | 必須 | キャッシュバックなどの支払内容     |
| キャンペーン名  | campaign_name          | 文字列    | 任意 | 対象となるキャンペーン名        |
| キャンペーン区分 | campaign_type          | 文字列    | 任意 | 通常、特別キャンペーンなどの区分    |
| 税区分      | tax_type               | 文字列    | 必須 | 税率10％、税考慮なしなどの区分    |
| 税率       | tax_rate               | 数値     | 任意 | 10％などの税率            |
| 備考       | notes                  | 文字列    | 任意 | 個別の注意事項や補足          |
| ステータス    | status                 | 文字列    | 必須 | 現在の作業状況             |
| 重複警告     | duplicate_warning      | 真偽値    | 必須 | 重複の可能性があるか          |
| 重複確認結果   | duplicate_check_result | 文字列    | 任意 | 重複なし、要確認、重複確定など     |
| 取込元ファイル名 | source_file_name       | 文字列    | 任意 | 取り込んだCSVまたはExcelの名前 |
| 取込元行番号   | source_row_number      | 整数     | 任意 | 元ファイル上の行番号          |
| 作成担当者    | created_by             | 担当者参照  | 任意 | 自動作成を実行した担当者        |
| 確認担当者    | checked_by             | 担当者参照  | 任意 | 作成結果を確認した担当者        |
| 取込日時     | imported_at            | 日時     | 必須 | データを取り込んだ日時         |
| 通知書作成日時  | document_created_at    | 日時     | 任意 | 支払通知書を作成した日時        |
| 確認日時     | checked_at             | 日時     | 任意 | 確認担当者が確認した日時        |
| 作成日時     | created_at             | 日時     | 必須 | アプリ上でレコードを作成した日時    |
| 更新日時     | updated_at             | 日時     | 必須 | 最後にデータを更新した日時       |

## 3. ステータス

支払対象データの作業状況を表す。

| ステータス | 内容                  |
| ----- | ------------------- |
| 未処理   | まだ取込・作成処理を開始していない   |
| 取込済み  | 一覧データを取り込んだ         |
| 作成済み  | 支払通知書を作成した          |
| 確認待ち  | 確認担当者による確認を待っている    |
| 確認済み  | 作成結果の確認が完了した        |
| 要修正   | 修正が必要と判定された         |
| エラー   | 処理を継続できない問題がある      |
| 完了    | 必要な作成・確認・修正がすべて完了した |

初期開発ではすべてのステータスを実装する必要はなく、業務フローの整理後に必要なものへ絞る。

---

## 4. 税区分

支払金額の税の扱いを表す。

| 税区分   | 内容             |
| ----- | -------------- |
| 税率10％ | 税率10％を考慮する     |
| 税考慮なし | 税金を考慮しない       |
| 要確認   | 税区分が不明または確認が必要 |

税区分の名称や計算方法は、実装前にサンプルデータを作成しながら調整する。

## 5. エラーデータ

取込、作成、確認時に発生したエラーを管理する。

| 項目名    | 英語名候補             | データ型候補 | 必須 | 内容               |
| ------ | ----------------- | ------ | -- | ---------------- |
| エラーID  | id                | 整数     | 必須 | エラーを識別する番号       |
| 支払対象ID | payment_record_id | 支払対象参照 | 必須 | エラーが発生した支払対象データ  |
| エラー種別  | error_type        | 文字列    | 必須 | 必須項目不足、形式不正、重複など |
| エラー内容  | error_message     | 文字列    | 必須 | 発生した問題の詳細        |
| 対象項目   | target_field      | 文字列    | 任意 | エラーが発生した項目       |
| エラー値   | error_value       | 文字列    | 任意 | 問題となった入力値        |
| 対応状況   | resolution_status | 文字列    | 必須 | 未対応、対応中、解消済み     |
| 対応内容   | resolution_note   | 文字列    | 任意 | 修正や確認の内容         |
| 対応担当者  | resolved_by       | 担当者参照  | 任意 | エラー対応を行った担当者     |
| 発生日時   | occurred_at       | 日時     | 必須 | エラーが発生した日時       |
| 解消日時   | resolved_at       | 日時     | 任意 | エラーを解消した日時       |
| 作成日時   | created_at        | 日時     | 必須 | エラーレコードの作成日時     |
| 更新日時   | updated_at        | 日時     | 必須 | 最後に更新した日時        |

---

## 6. エラー種別

初期開発では、以下のエラー種別を想定する。

| エラー種別    | 内容                  |
| -------- | ------------------- |
| 必須項目不足   | 会社名や支払金額などが未入力      |
| データ形式不正  | 金額や日付の形式が正しくない      |
| 発行番号重複   | 同じ発行番号がすでに登録されている   |
| 重複候補     | 会社名、金額、支払日などが一致している |
| 税区分未設定   | 税区分が入力されていない        |
| 通知書作成エラー | 支払通知書の作成処理に失敗した     |
| 確認不一致    | 元データと作成結果に違いがある     |
| その他      | 上記に該当しないエラー         |

## 7. 作業ログデータ

誰が、いつ、何を行ったかを記録する。

| 項目名    | 英語名候補             | データ型候補 | 必須 | 内容              |
| ------ | ----------------- | ------ | -- | --------------- |
| ログID   | id                | 整数     | 必須 | 作業ログを識別する番号     |
| 支払対象ID | payment_record_id | 支払対象参照 | 任意 | 操作対象となった支払対象データ |
| 操作者    | operator          | 担当者参照  | 任意 | 操作を行った担当者       |
| 操作種別   | action_type       | 文字列    | 必須 | 取込、作成、確認、修正など   |
| 操作内容   | action_detail     | 文字列    | 必須 | 実行した操作の詳細       |
| 対象項目   | target_field      | 文字列    | 任意 | 変更した項目          |
| 変更前の値  | old_value         | 文字列    | 任意 | 修正前の値           |
| 変更後の値  | new_value         | 文字列    | 任意 | 修正後の値           |
| 操作日時   | created_at        | 日時     | 必須 | 操作を実行した日時       |

---

## 8. 操作種別

作業ログには、以下の操作を記録する。

| 操作種別    | 内容                |
| ------- | ----------------- |
| データ取込   | CSVまたはExcelを取り込んだ |
| データ修正   | 支払対象データを修正した      |
| 通知書作成   | 支払通知書を作成した        |
| 確認結果登録  | 確認済み、要修正などを登録した   |
| ステータス変更 | 作業ステータスを変更した      |
| エラー登録   | 新しいエラーを登録した       |
| エラー対応   | エラーの対応内容を登録した     |
| エラー解消   | エラーを解消済みにした       |

---

## 9. 担当者データ

初期開発では、作成担当者や確認担当者を記録するために使用する。

| 項目名   | 英語名候補      | データ型候補 | 必須 | 内容               |
| ----- | ---------- | ------ | -- | ---------------- |
| 担当者ID | id         | 整数     | 必須 | 担当者を識別する番号       |
| 担当者名  | name       | 文字列    | 必須 | 画面に表示する名前        |
| 役割    | role       | 文字列    | 必須 | 管理者/SV、作成担当、確認担当 |
| 有効状態  | is_active  | 真偽値    | 必須 | 現在利用できる担当者か      |
| 作成日時  | created_at | 日時     | 必須 | 担当者を登録した日時       |
| 更新日時  | updated_at | 日時     | 必須 | 最後に更新した日時        |

初期開発でログイン機能を実装しない場合でも、担当者を選択するための簡易データとして使用する。

## 10. 支払通知書ファイルデータ

作成された支払通知書ファイルを管理する。

| 項目名     | 英語名候補             | データ型候補 | 必須 | 内容             |
| ------- | ----------------- | ------ | -- | -------------- |
| ファイルID  | id                | 整数     | 必須 | ファイルを識別する番号    |
| 支払対象ID  | payment_record_id | 支払対象参照 | 必須 | 対象となる支払データ     |
| ファイル名   | file_name         | 文字列    | 必須 | 作成したファイル名      |
| ファイル保存先 | file_path         | 文字列    | 必須 | ファイルの保存場所      |
| ファイル形式  | file_type         | 文字列    | 必須 | Excel、PDFなど    |
| バージョン   | version           | 整数     | 必須 | 再作成した場合の版数     |
| 作成担当者   | created_by        | 担当者参照  | 任意 | ファイル作成を実行した担当者 |
| 作成日時    | created_at        | 日時     | 必須 | ファイルを作成した日時    |

## 12. 今後の設計で決めること

以下の項目は、データベース設計や実装の段階で決定する。

* 支払通知書をExcelとPDFのどちらで出力するか
* 税率を数値として保存するか
* キャンペーン情報を別データとして管理するか
* 発行番号をどのように採番するか
* 同一会社への複数支払をどのように扱うか
* 重複判定に使用する項目の組み合わせ
* 作成担当者と確認担当者を必ず別人にするか
* ファイルをアプリ内に保存するか、保存先のみを記録するか
