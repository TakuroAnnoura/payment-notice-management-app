# MVP総合テスト計画 v1

## 1. 目的

本書では、支払通知書作成・進捗管理アプリのMVPが、仕様どおりに動作することを確認する。

以下の基本業務フローを、正常系・異常系の両方でテストする。

```text
CSV取込
    ↓
入力内容・形式・重複チェック
    ↓
支払通知書作成
    ↓
確認担当者による確認
    ↓
要修正または確認済み
    ↓
修正・再作成・再確認
    ↓
管理者/SVによる完了処理
```

---

## 2. テスト結果の記録方法

各テストは、以下の結果で記録する。

| 結果  | 内容           |
| --- | ------------ |
| 未実施 | まだテストしていない   |
| OK  | 想定どおり動作した    |
| NG  | 想定と異なる動作をした  |
| 保留  | 仕様確認や追加対応が必要 |

NGまたは保留の場合は、備考欄に内容を記録する。

## 12. テスト時に確認する追加課題

総合テスト中に、以下の点も確認する。

* 100件以上の一覧でも画面が見づらくならないか
* 長い会社名や住所がレイアウトを崩さないか
* 長い確認コメントが一覧を圧迫しないか
* 同じCSVを再度取り込んだ場合の挙動
* 支払通知書の金額・日付の配置
* ステータス名やボタン文言が分かりやすいか
* 一般画面だけで主要業務を完結できるか
* 管理画面でしかできない操作が残っていないか
* 1人が作成・確認の複数権限を持てるようにし、当該データの作成者本人だけを確認候補から除外する
* 支払通知書へ出力する備考と、社内連携用の内部メモを分離する
* エラー解消後の対応結果を「修正版待ち・再取込・対象外・重複除外・継続」などに分けて管理する
* ダッシュボードでは、業務進捗のカード群と要対応エラーを視覚的に分離する
* 「取込済み」「対象外」を追加し、全体件数の内訳を把握できるようにする

## 13. MVP総合テスト完了条件

以下を満たした場合、MVP総合テスト完了とする。

* 基本業務フローを最初から最後まで実行できる
* 重大なNG項目が残っていない
* エラーや入力不正を適切に停止できる
* 修正履歴と作業履歴を確認できる
* 支払通知書を作成・再作成・ダウンロードできる
* 管理者/SVが完了まで管理できる
* 保留項目がMVP対象外または今後の拡張として整理されている
