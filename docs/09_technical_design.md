# 技術構成・開発方針 v1

## 1. 目的

本書では、支払通知書作成・進捗管理アプリを実装するために使用する技術と、初期開発の進め方を整理する。

初期開発では、限られた開発期間内にMVPを完成させることを優先し、必要以上に構成を複雑にしない。

---

## 2. 初期開発の技術構成

| 区分         | 使用技術候補               | 用途               |
| ---------- | -------------------- | ---------------- |
| 開発言語       | Python               | アプリ全体の処理         |
| Webフレームワーク | Django               | 画面、データベース操作、業務処理 |
| フロントエンド    | HTML、CSS、Bootstrap   | 画面表示とレイアウト       |
| データベース     | SQLite               | 支払対象、エラー、作業ログの保存 |
| CSV処理      | Python標準csvまたはpandas | 支払対象者一覧の取込       |
| Excel処理    | openpyxl             | 支払通知書テンプレートへの書込み |
| バージョン管理    | Git、GitHub           | ソースコードと仕様書の管理    |
| 開発環境       | Visual Studio Code   | 仕様書作成とプログラム開発    |

---

## 3. アプリの基本構成

初期開発では、Djangoで画面表示、業務処理、データベース操作をまとめて実装する。

```text
ブラウザ
   ↓
Djangoの画面
   ↓
CSV取込・チェック・通知書作成処理
   ↓
Djangoモデル
   ↓
SQLite
```

Reactを使用してフロントエンドとバックエンドを分離する構成は、MVP完成後の拡張候補とする。

---

## 4. Djangoを中心にする理由

初期開発でDjangoを中心にする理由は以下のとおりである。

* PythonでCSVやExcelの処理を実装しやすい
* Djangoの機能で画面とデータベースを一体的に作成できる
* 管理画面を利用して、開発途中のデータを確認できる
* ログイン機能を後から追加しやすい
* ReactとAPIを同時に開発する場合より、実装範囲を抑えられる
* 限られた開発期間内にMVPを完成させやすい

---

## 5. フロントエンドの方針

初期開発では、Djangoテンプレートを使用してHTML画面を作成する。

画面の見た目については、Bootstrapを利用して以下を整える。

* ナビゲーション
* 一覧表
* 入力フォーム
* ステータス表示
* エラー表示
* ダッシュボードの件数カード
* ボタンや確認メッセージ

高度なデザインよりも、業務で迷わず操作できる分かりやすさを優先する。

---

## 6. データベースの方針

初期開発ではSQLiteを使用する。

SQLiteはDjangoの初期設定で利用でき、別途データベースサーバーを構築せずに開発を開始できる。

MVP完成後、本番運用を想定する場合はMySQLやPostgreSQLへの移行を検討する。

---

## 7. 一覧データ取込の方針

MVPではCSVファイルの取込を必須とする。

取込時には以下の処理を行う。

1. CSVファイルを選択する
2. ファイルの列名を確認する
3. 各行のデータを読み込む
4. 必須項目を確認する
5. 金額や日付の形式を確認する
6. 発行番号の重複を確認する
7. 正常、警告、エラーに分類する
8. 正常データをデータベースへ保存する

Excelファイルの直接取込は、MVP完成後の拡張候補とする。

---

## 8. 支払通知書作成の方針

初期開発では、Excel形式の支払通知書テンプレートを使用する。

Pythonのopenpyxlを利用し、テンプレート内の指定セルへ以下の情報を書き込む。

* 発行番号
* 会社名
* 住所
* 事業者登録番号
* 支払金額
* 支払日
* 内訳
* 税区分

作成したExcelファイルは、画面からダウンロードまたは確認できるようにする。

PDF出力は、Excel形式での作成が完成した後の拡張候補とする。

---

## 9. 初期開発でReactを使用しない理由

Reactを使用する場合、以下の追加作業が必要になる。

* Django側のAPI作成
* React側の画面作成
* APIとの通信処理
* DjangoとReactの開発環境管理
* エラー処理や認証方法の設計

初期開発ではMVP完成を優先し、Djangoテンプレートで画面を作成する。

Reactは、MVP完成後にダッシュボードや一覧画面を置き換える形で利用を検討する。

---

## 10. 開発の基本方針

開発は以下の方針で進める。

* 一度にすべての機能を作らない
* 小さな単位で実装し、画面で動作確認する
* 正常データだけでなく、エラーデータでも確認する
* 仕様を変更した場合はdocsの内容も更新する
* MVPに不要な機能は後回しにする
* Gitで作業単位ごとに変更履歴を残す

---

## 11. 実装順序

以下の順番で実装する。

1. Djangoプロジェクトを作成する
2. Djangoアプリを作成する
3. データベースモデルを作成する
4. Django管理画面でデータを確認できるようにする
5. サンプルCSVを作成する
6. CSV取込機能を作成する
7. 必須項目・形式・重複チェックを作成する
8. 支払対象者一覧画面を作成する
9. 支払対象者詳細画面を作成する
10. 支払通知書のExcel作成機能を作成する
11. 確認・ステータス更新機能を作成する
12. エラー管理機能を作成する
13. 作業ログ機能を作成する
14. ダッシュボードを作成する
15. テストデータで一連の業務フローを確認する
16. READMEと画面キャプチャを整備する

---

## 12. MVP完成後の技術的な拡張候補

* Reactによる画面の再構築
* Django REST FrameworkによるAPI化
* MySQLまたはPostgreSQLへの移行
* PDF形式での支払通知書出力
* Excelファイルの取込
* 公的CSVの自動取得
* 定期処理の実装
* メール送付機能
* ログインと権限管理
* 自動テストの追加
* クラウド環境への公開

---

## 13. 初期開発の技術構成まとめ

MVPでは、以下の構成を基本とする。

```text
Python
Django
HTML / CSS
Bootstrap
SQLite
CSV
openpyxl
Git / GitHub
Visual Studio Code
```

まずはこの構成で、CSV取込から支払通知書作成、確認、進捗管理までの一連の業務を完成させる。
