# 支払通知書テンプレート仕様 v1

## 1. 目的

本書では、本アプリから自動作成する支払通知書の記載項目、固定文言、差込項目、配置方針を整理する。

初期開発では、実在する企業の正式帳票を完全に再現するのではなく、支払通知書の自動作成機能を確認できる開発用テンプレートを作成する。

実務で使用する場合は、経理・法務などから提供された正式なひな型をもとに、項目、文言、レイアウト、セル位置を調整する。

---

## 2. 初期開発の出力形式

初期開発では、支払通知書をExcel形式で出力する。

| 項目        | 内容                                                |
| --------- | ------------------------------------------------- |
| ファイル形式    | Excel（`.xlsx`）                                    |
| 使用ライブラリ   | openpyxl                                          |
| テンプレート保存先 | `document_templates/payment_notice_template.xlsx` |
| 作成ファイル保存先 | `media/generated_documents/支払対象ID/`               |
| 用紙イメージ    | A4縦                                               |
| シート名      | 支払通知書                                             |

PDF形式での出力は、Excel作成機能の完成後に検討する。

---

## 3. テンプレートの基本方針

支払通知書には、以下の2種類の情報を記載する。

### 3.1 支払対象データから差し込む情報

CSVから取り込み、データベースへ保存した値を使用する。

* 発行番号
* 発行日
* 会社名
* 住所
* 事業者登録番号
* 支払金額
* 支払日
* 内訳
* 税区分
* 備考

### 3.2 テンプレートへ固定で記載する情報

支払対象ごとに変更しない文言として、テンプレート側へ記載する。

* 「支払通知書」というタイトル
* 支払いを通知する案内文
* 振込先の表示
* 発行元会社名
* 発行元住所
* 問い合わせ先
* 注意事項

---

## 4. 発行日の扱い

支払通知書に記載する日付は、「作成日時」ではなく「発行日」とする。

MVPでは、支払通知書を作成した日を発行日として使用する。

```text
発行日 ＝ 支払通知書を作成した日
```

Excelには時刻を表示せず、日付のみを記載する。

表示形式は以下とする。

```text
2026年6月15日
```

将来的に作成日と発行日を分ける必要が生じた場合は、`issue_date`を独立したデータ項目として追加する。

---

## 5. 振込先情報の扱い

初期開発では、銀行名、支店名、口座種別、口座番号などの具体的な振込先情報をCSV、データベース、支払通知書へ保存しない。

支払通知書には、以下の固定文言を記載する。

```text
振込先：ご登録振込先
```

これは、メール送信やファイル共有時に口座情報が漏えいするリスクを抑えるための業務上の配慮である。

将来的に口座情報を扱う場合は、閲覧権限、暗号化、送付方法、保存期間などを別途設計する。

---

## 6. 記載項目

### 6.1 ヘッダー情報

| 表示項目   | 種類 | 内容           |
| ------ | -- | ------------ |
| 帳票タイトル | 固定 | 支払通知書        |
| 発行日    | 差込 | 支払通知書を作成した日  |
| 発行番号   | 差込 | 支払通知書を識別する番号 |
| 発行元会社名 | 固定 | 開発用の仮会社名     |
| 発行元住所  | 固定 | 開発用の仮住所      |
| 問い合わせ先 | 固定 | 開発用の仮問い合わせ先  |

### 6.2 支払先情報

| 表示項目    | 種類 | 内容                 |
| ------- | -- | ------------------ |
| 会社名     | 差込 | 支払先の会社名            |
| 住所      | 差込 | 支払先の住所             |
| 事業者登録番号 | 差込 | Tと13桁の数字で構成される登録番号 |

会社名の末尾には、帳票上で「御中」を付けることを想定する。

CSVやデータベースには「御中」を含めず、テンプレート出力時に付加する。

### 6.3 支払情報

| 表示項目 | 種類 | 内容              |
| ---- | -- | --------------- |
| 支払金額 | 差込 | 支払予定金額          |
| 支払日  | 差込 | 支払予定日または支払日     |
| 振込先  | 固定 | ご登録振込先          |
| 内訳   | 差込 | キャッシュバックなどの支払内容 |
| 税区分  | 差込 | 税率10％、税考慮なしなど   |
| 備考   | 差込 | 個別の注意事項や補足      |

---

## 7. 固定文言

開発用テンプレートでは、以下の文言を使用する。

### 7.1 通知文

```text
下記のとおりお支払い予定をご通知いたします。
内容をご確認くださいますようお願いいたします。
```

### 7.2 振込先

```text
ご登録振込先
```

### 7.3 注意事項

```text
本書は開発用のサンプル帳票です。
実際の支払内容については、発行元へお問い合わせください。
```

### 7.4 発行元情報

MVPでは、実在しない仮の情報を使用する。

```text
発行元：株式会社サンプル
所在地：福岡県福岡市〇〇区〇〇
問い合わせ先：sample@example.com
```

実在する会社名、住所、メールアドレス、電話番号は使用しない。

## 8. セル配置案

初期開発では、以下のセル配置を使用する。

| 項目      | ラベルセル   | 値を入れるセルまたは範囲 |
| ------- | ------- | ------------ |
| タイトル    | ―       | A1:F1        |
| 発行日     | A3:B3   | C3:F3        |
| 発行番号    | A4:B4   | C4:F4        |
| 会社名     | A6:B6   | C6:F6        |
| 住所      | A7:B7   | C7:F7        |
| 事業者登録番号 | A8:B8   | C8:F8        |
| 通知文     | ―       | A10:F11      |
| 支払金額    | A13:B13 | C13:F13      |
| 支払日     | A14:B14 | C14:F14      |
| 振込先     | A15:B15 | C15:F15      |
| 内訳      | A17:B17 | C17:F19      |
| 税区分     | A20:B20 | C20:F20      |
| 備考      | A22:B22 | C22:F24      |
| 注意事項    | ―       | A26:F27      |
| 発行元情報   | ―       | D29:F32      |

セルの結合範囲、行の高さ、列幅、文字サイズなどは、テンプレート作成時に調整する。

正式なひな型を使用する場合は、そのひな型のセル位置に合わせて本仕様を更新する。

## 9. 表示形式

### 9.1 支払金額

支払金額は、3桁区切りと円表記で表示する。

```text
110,000円
```

小数は表示しない。

### 9.2 日付

発行日と支払日は、次の形式で表示する。

```text
2026年7月31日
```

### 9.3 事業者登録番号

事業者登録番号が登録されている場合は、その値を表示する。

登録されていない場合は、次のように表示する。

```text
未登録
```

### 9.4 税区分

データベースに保存された値を、画面表示用の日本語へ変換して表示する。

| 保存値                | 帳票上の表示 |
| ------------------ | ------ |
| taxable_10         | 税率10％  |
| non_taxable        | 税考慮なし  |
| needs_confirmation | 要確認    |

---

## 10. ファイル名

作成するファイル名は、以下の形式とする。

```text
発行番号_vバージョン番号.xlsx
```

例：

```text
CB-2026-101_v1.xlsx
CB-2026-101_v2.xlsx
```

ファイル名に使用できない文字が含まれている場合は、アンダースコアへ置き換える。

---

## 11. バージョン管理

支払通知書を再作成した場合は、既存ファイルを上書きせず、新しいバージョンとして保存する。

```text
v1：初回作成
v2：修正後の再作成
v3：再修正後の作成
```

最新バージョンを現行版とし、それ以前のファイルは旧版として履歴を残す。

---

## 12. 作成を停止する条件

以下に該当する場合は、支払通知書を作成しない。

* 必須項目が不足している
* 支払金額が正しい数値ではない
* 支払日が正しい日付ではない
* 税区分が未設定または不正である
* 発行番号の重複エラーがある
* 未解消の重大なエラーがある
* 重複警告が未確認である

---

## 13. テンプレートの変更方針

テンプレートのレイアウトや固定文言は、今後変更する可能性がある。

変更時は、以下を確認する。

1. テンプレートのセル位置を変更する
2. 本仕様書のセル配置表を更新する
3. `document_generator.py`の書込先セルを更新する
4. 正常データで出力確認を行う
5. 長い会社名、住所、内訳でも表示が崩れないか確認する
6. 再作成時のバージョン管理を確認する

---

## 14. MVPの受け入れ条件

以下を満たした場合、支払通知書テンプレートのMVPが完成したものとする。

* 発行日が作成日の日付で表示される
* 発行番号が正しく表示される
* 会社名、住所、事業者登録番号が正しく表示される
* 支払金額が3桁区切りと円表記で表示される
* 支払日が日本語の日付形式で表示される
* 振込先に「ご登録振込先」と表示される
* 内訳、税区分、備考が正しく表示される
* 仮の発行元情報が表示される
* Excelファイルとして保存・ダウンロードできる
* 再作成した場合にバージョン番号が増加する
* 過去のバージョンが削除されずに残る

## 16. 税額内訳の扱い

MVPでは、支払金額を一覧データから受け取った支払総額として表示し、税区分を併記する。

将来的には、以下の項目を分けて管理・表示する。

- 税抜金額
- 消費税額
- 支払合計額

税額を自動計算する場合は、一覧の金額が税込・税抜のどちらであるか、および端数処理のルールを事前に定義する。
